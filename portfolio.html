Gargantuan FM

Gargantuan.FM

Visit Project | View On Github

The Project.

With Gargantuan.FM, the user can easily throw on a curated playlist (channel) sourced from SoundCloud or he/she can make their own right from the site. While the music plays, the user also has the ability to change the visual scenery, sourced from curated YouTube playlists.

I worked with Alan Chu & Alisa Schink on this application for our Front End Development Project for Rutgers Coding Bootcamp.

This project was inspired by Poolside.fm and a desire to build something that can be both fun and useful.

 

The Technologies.

  • HTML
  • Foundation CSS Framework
  • JavaScript
  • jQuery
  • Moment.js
  • Audio.js
  • Firebase
  • SoundCloud API
  • YouTube API
  • Google Fonts

Train Scheduler

Train Scheduler

Visit Web Site | View On Github

The Project.

For our Week 7 homework in Rutgers Coding Bootcamp, we were tasked with creating a Train Schedule application that allows the user to create, edit, or delete a train by entering its name, destination, first time it runs, and frequency. A bonus task was to include user authentication through Firebase in order to see the page's content. I chose to use Google authentication.

All of the train information was stored in Firebase and displayed on the page with the time automatically updating to indicate how many minutes away the next train is and what time it will be arriving.

 

The Technologies.

  • HTML
  • Twitter Bootstrap
  • JavaScript
  • jQuery
  • Moment.js
  • Firebase
  • Google Authentication (with Firebase)

Jenkinson's Boardwalk

Jenkinson's Boardwalk

Visit Web Site

The Project.

Over the past ten years, I've been lucky enough to play a part in building the Jenkinson's Boardwalk web site twice. The first time in Drupal, the next in WordPress.

The decision to switch to WordPress made sense because of how hands-on the Jenkinson's staff needs to be on a daily basis with pushing new information onto their site for visitors. Sometimes Drupal is a difficult backend to learn for people juggling a lot of various non-web related tasks, but there was a need for the three main departments of the business to access their part of the site without running the risk of disrupting one another.

Using WordPress Multsite, we were able to build a new site in three sections for Jenkinson's that allowed each department to conduct routine updates on their specific areas of the site without ever having to worry about anything the other departments were doing.

 

The Technologies.

  • WordPress Multisite
  • WooCommerce
  • Visual Composer
  • Calendarize.it
  • Authorize.net
